Built with Next.js 14 App Router, deployed on Vercel, and way more interesting under the hood than it looks.

The now-playing widget in the nav is the most involved part — Apple Music doesn't have a server-side "is playing right now" endpoint, so I built a Redis-backed heuristic: when a new song appears in the recently played API, timestamp it and compare elapsed time against the track's duration. If you're within the window, you're probably still listening. MusicKit JS on the client can upgrade the status if it detects active playback.

The travel section is MDX-based — each trip is a markdown file with frontmatter for metadata and a body for the write-up. Same pattern now used for projects.

Cloudflare sits in front of Vercel for caching and bot protection. Custom robots.txt blocks most AI crawlers. There's a honeypot route that serves fake credentials and infinite pagination links to waste scrapers' time.

The font stack is all system fonts — no external font loading, which keeps CLS low and first paint fast.
